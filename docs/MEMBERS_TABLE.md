# メンバー表 仕様

## 概要

メンバー表ページで表示するメンバー情報の定義と、権限による表示の出し分けルール。

## データ項目

| 項目 | 型 | 管理者 | メンバー | 備考 |
|------|-----|:------:|:--------:|------|
| フルネーム | string | ✓ | ✓ | fullName。未設定時は displayName を使用 |
| 電話番号 | string | ✓ | - | 個人情報 |
| メールアドレス | string | ✓ | - | 個人情報 |
| 住所 | string | ✓ | - | 個人情報 |
| 身長 | number (cm) | ✓ | ✓ | |
| ポジション | string | ✓ | ✓ | PG, SG, SF, PF, C など |
| ユニフォーム番号 | number[] | ✓ | ✓ | 複数可（ホーム/アウェイ等） |
| 参加率 | number (%) | ✓ | - | 活動＋提案への YES/MAYBE 回答率。管理者のみ |

## 権限

- **OWNER（管理者）**: 全項目を閲覧・編集可能
- **MEMBER（一般メンバー）**: 電話番号・メール・住所を除く項目のみ閲覧可能（編集は自分のみ or 不可）

## 関連ファイル

- `prisma/schema.prisma` - User, UserUniform モデル
- `lib/members.ts` - 型定義・定数

## 追加検討機能の提案

サークル管理で便利になりそうな機能のアイデアです。

### 1. 出欠率・参加率の表示
- 活動日程・交流戦提案への出欠を集計し、参加率を表示
- 既存の `ActivityAttendance`, `Availability` を活用可能
- 「どのメンバーがよく来るか」の可視化で、連絡や役割分担の参考に

### 2. 在籍ステータス
- `status`: ACTIVE（活動中） / INACTIVE（休部） / RETIRED（退部）
- 名簿に残しつつ「現在活動中」のメンバーだけフィルタ表示
- 退部者も試合記録の紐づけに残せる

### 3. 緊急連絡先
- 管理者用の emergencyContact, emergencyPhone
- 怪我や事故時の連絡に使用

### 4. ユニフォームサイズ
- 注文時の参考（S, M, L, XL 等）
- ユニフォーム番号とあわせて管理すると便利

### 5. 誕生日
- お祝いやイベント企画の参考
- 表示はメンバー全員 or 管理者のみなどルールを決めて運用

### 6. 備考（管理者用メモ）
- 自由記述の memo フィールド
- アレルギーや持病など、管理者だけが知っておきたい情報

### 7. 入部日
- 既存の `createdAt` を「登録日」として表示
- 必要なら `joinedAt` を別途持って入部日を管理

### 8. プロフィール画像
- 名簿で顔と名前を対応させやすくする
- 既存の Google アカウントの写真を表示する選択肢もあり
