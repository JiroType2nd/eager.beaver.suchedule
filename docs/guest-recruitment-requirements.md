# ゲスト募集機能 要件定義書

## 1. 概要

### 1.1 背景
- 自チームが確保した体育館での活動、および交流戦で相手チームの体育館へのお邪魔する日程調整機能は既に実装済み
- 本機能は、**他サークルからゲストとしてお誘いを受けた場合**の日程調整に対応する

### 1.2 目的
他サークルからのゲスト募集に参加するための募集情報の登録・管理を行う。募集者は「ゲスト募集」として1回の募集を作成し、定員に達した時点で募集を終了する。

---

## 2. ユーザーストーリー

**作為:** チームの管理者・スタッフ  
**目的:** 他サークルからゲストとして招待された際に、チームメンバーへ参加を募集したい  
**結果:** 活動タブから「ゲスト募集を行う」を選び、募集内容（開催場所・レベル・定員・参加費など）を登録できる

---

## 3. 機能要件

### 3.1 エントリーポイント（画面遷移）

| 項目 | 内容 |
|------|------|
| **場所** | 活動タブ（`/activities`） |
| **追加要素** | 「ゲスト募集を行う」ボタン |
| **配置** | 活動日程一覧画面上部のボタン群に追加。既存の「登録」「一括編集」等と並列に配置 |
| **遷移先** | ゲスト募集 新規登録画面（例: `/activities/guest-recruitment/new`） |

**参考（現行の活動ページ構成）:**
- 一括編集 / 一括出欠 / カレンダー / カレンダーに一括追加 / **登録** ← ここに「ゲスト募集を行う」を追加

### 3.2 ゲスト募集 登録フォーム

活動日程登録と同様のUI・UXを基本とし、以下の項目を入力する。

| No | 項目 | ラベル | 入力形式 | 必須 | 備考 |
|----|------|--------|----------|------|------|
| 1 | 実施場所 | 実施場所（または 開催場所） | セレクト or テキスト | ✓ | 既存の `VENUE_OPTIONS` を流用し、選択または「その他」で自由入力 |
| 2 | レベル | レベル | セレクト（単一選択） | ✓ | 後述のレベル選択肢 |
| 3 | 定員 | 定員 | 数値入力 | ✓ | 正の整数。満員になった時点で募集終了 |
| 4 | 参加費 | 参加費 | 数値入力（円表示） | - | 数値入力で「○○円」と表示。0の場合は「無料」等と表示可 |
| 5 | 日程 | 日程 | 日付 | ✓ | 活動日程登録と同様 |
| 6 | 開始時刻 | 開始時刻 | 時刻セレクト | ✓ | `TIME_OPTIONS` を流用 |
| 7 | 終了時刻 | 終了時刻 | 時刻セレクト | ✓ | `TIME_OPTIONS` を流用 |
| 8 | タイトル | タイトル | テキスト | - | 任意（例: ○○サークル主催 ゲスト募集） |
| 9 | 備考 | 備考 | テキストエリア | - | 任意 |

### 3.3 レベル選択肢

以下の4種を選択肢として用意する。

| 値 | 表示名 |
|----|--------|
| ダンジョンミックス | ダンジョンミックス |
| エンジョイ | エンジョイ |
| 男性のみ | 男性のみ |
| しっかりめ | しっかりめ |

### 3.4 募集の終了条件

- **定員に達した時点で募集終了**
- 定員は1回限り（追加募集なし）
- 募集終了後は、参加申込の受付を停止する

### 3.5 一覧表示（活動タブ内）

- ゲスト募集は通常の活動日程と**混ぜて表示**する
- ゲスト募集であることを視覚的に区別できるようにする
  - **色を変える**（例: 背景色・枠色・バッジ色の変更）
  - ラベルやバッジで「ゲスト募集」と明示する

### 3.6 参加申込の受付

- **チームメンバー（ログインユーザー）のみ**が申込可能
- 外部（他サークルの人など）からの申込は受付しない

---

## 4. 非機能要件（案）

- 認証: ログインユーザーのみアクセス可能
- 権限: ゲスト募集の作成は OWNER / STAFF に限定するか、MEMBER も可にするか要検討
- レスポンシブ: 既存アプリと同様、モバイルファースト
