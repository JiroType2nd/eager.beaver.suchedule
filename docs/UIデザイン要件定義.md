# UIデザイン要件定義

交流戦スケジュールアプリのモダンで直感的なUIを実現するための要件定義です。  
チームテーマカラー（ネイビー）を基調とし、アイコンを多用した図解的なUIを目指します。

---

## 1. 配色カラーパレット

添付いただいた名刺デザインを参考にした配色です。

| 用途 | 色名 | 色コード例 | 使い方 |
|------|------|-----------|--------|
| **メイン背景** | 濃いネイビー | `#0f172a`〜`#1e3a5f` | メイン画面・カード背景 |
| **テキスト** | 白 | `#ffffff` | メインの文字色 |
| **サブテキスト** | 明るいグレー | `#94a3b8` | 補足説明・日付・場所など |
| **アクセント** | ゴールド | `#c9a227`〜`#d4af37` | CTAボタン・ロゴ・強調部分 |
| **成功/確定** | ゴールド系グリーン | `#22c55e` | 確定済み・参加OKなど |
| **注意/募集中** | アンバー | `#f59e0b` | 募集中・未確定などの状態 |
| **リンク/ホバー** | 明るいネイビー | `#334155` | ホバー時・セカンダリ背景 |

**補足:**
- ネイビー背景上では白テキストで視認性を確保
- ゴールドは「確定」「重要」「アクション」など限定的に使用
- カードやセクションの区切りには軽いグラデーションや影を活用

## 2. アイコン活用方針

**原則: テキストの前にアイコンを置き、一見で意味が分かるようにする**

### 2.1 画面・セクションごとのアイコン提案

| 場所 | アイコン例 | 意味 |
|------|-----------|------|
| ホーム | ホーム / ダッシュボード | トップ画面 |
| 活動日程 | 📅 カレンダー | 日程一覧 |
| 次回イベント（交流戦） | 🏀 バスケ / vs アイコン | 交流戦イベント |
| 募集中の提案 | 🔔 ベル / 募集中バッジ | 対応が必要 |
| 未決定の枠 | 📋 クリップボード | 候補枠 |
| 場所 | 📍 ピン | 会場・住所 |
| 日時 | 🕐 時計 / カレンダー | 日付・時刻 |
| メンバー | 👥 グループ | チームメンバー |
| マイページ | 👤 ユーザー | 自分のプロフィール |
| 試合記録 | 🏆 トロフィー / スコアボード | 試合履歴 |
| ログアウト | 🚪 出口 | サインアウト |

### 2.2 状態・アクションのアイコン

| 状態 | アイコン例 | 使い方 |
|------|-----------|--------|
| 確定 | ✓ チェック | 確定済み・参加OK |
| 募集中 | ⏳ 砂時計 | 出欠収集中 |
| 双方OK | 🤝 握手 | 確定待ち |
| 地図 | 🗺 地図 | 会場へのルート |
| 編集 | ✏ ペン | 編集ボタン |
| 追加 | + / ➕ | 新規作成ボタン |

**アイコンライブラリ案:** Lucide Icons または Heroicons（軽量・無料・Tailwindと相性◎）

## 3. 直感的なUIデザイン原則

### 3.1 カード型レイアウト

- 各イベント・提案・枠は**カード**で表示
- カード左端に**色付きバー**や**アイコン**で種別・状態を表現
  - 例: 募集中 → アンバー系 / 確定 → ゴールド系 / 活動 → ネイビー系
- タップで詳細へ遷移することを示す（ホバー時・タップ時の見た目の変化）

### 3.2 情報の階層

- **タイトル（イベント名・相手チームなど）** … 大きく、太字
- **日時** … アイコン＋テキスト、サブテキスト色
- **場所** … アイコン＋テキスト、サブテキスト色
- **状態バッジ** … 小さなピル型、右端や下部に配置

### 3.3 ステータス表示

状態を色とアイコンで即座に判断できるようにする:

| 状態 | 表現例 |
|------|--------|
| 出欠収集中 | アンバー背景 + 砂時計アイコン + 「募集中」 |
| 双方OK・確定待ち | ゴールド背景 + 握手アイコン + 「確定待ち」 |
| 確定済み | 緑系背景 + チェックアイコン + 「確定」 |
| 中止 | グレー + 取り消し線 |

### 3.4 ナビゲーション（BottomNav）

- テキストだけでなく**各項目にアイコン**を表示
- アクティブ時: ゴールドまたは白で強調
- 非アクティブ時: サブテキスト色
- 背景はネイビーまたは半透明ネイビーで全体のテーマと統一

## 4. コンポーネント別のデザイン方針

### 4.1 ボタン

| 種類 | デザイン |
|------|----------|
| プライマリ（ログイン・確定など） | ゴールド背景、白文字、角丸 |
| セカンダリ（キャンセル・戻る） | 枠線のみ、白またはゴールドの枠 |
| 危険（中止・削除） | 赤系の枠線、または赤背景 |

### 4.2 入力フォーム

- 入力欄はネイビー背景上で**白または薄いグレー背景**の枠
- ラベルはアイコン＋テキスト
- 必須項目は*またはラベルにバッジ

### 4.3 空状態（データなし）

- アイコン（大きめ）＋「〇〇はありません」＋必要なら「〇〇を追加」リンク
- 例: カレンダーアイコン + 「活動日程はありません」+ 「活動を作成」

### 4.4 FAB（フローティングアクションボタン）

- 枠作成などのメインアクション
- ゴールド背景、白の「+」アイコン
- 右下固定、ボトムナビの上に配置

---

## 5. レスポンシブ・スマホ優先

- **スマホファースト**（既存方針を維持）
- カードは幅いっぱい、余白は`px-4`程度
- タッチターゲットは最低44×44px
- セーフエリア（ノッチ・ホームインジケータ）を考慮した`pb-safe`継続

---

## 6. 実装の優先順位（提案）

1. **Phase 1: 配色・テーマの適用**
   - `tailwind.config.ts` にネイビー・ゴールドのカラーを追加
   - `globals.css` のデフォルト背景・テキスト色をネイビーテーマに変更
   - ダークテーマをデフォルトとして固定（または選択式）

2. **Phase 2: アイコン導入**
   - Lucide React または Heroicons をインストール
   - BottomNav、ホーム画面のセクション見出し、カードにアイコン追加

3. **Phase 3: カード・コンポーネントのスタイル統一**
   - カードの背景・ボーダー・影をネイビーテーマに合わせて更新
   - 状態バッジ（募集中・確定待ち等）をアイコン＋色で表現

4. **Phase 4: ボタン・フォームのスタイル統一**
   - プライマリ・セカンダリボタンのデザイン統一
   - フォーム入力欄のスタイル更新

---

## 7. 参考：名刺デザインのポイント（添付画像より）

- ネイビー背景 + 白テキスト + ゴールドアクセント
- 電話・メール・住所にはそれぞれ専用アイコンを配置
- サンセリフ系のクリーンなフォント
- 余白を活かしたミニマルなレイアウト
- アイコンとテキストの左揃えで整然とした印象

---

## 8. 補足・今後の検討事項

- **フォント:** 日本語はNoto Sans JP、英語・数字はInter など、読みやすいサンセリフを推奨
- **アニメーション:** カードタップ時の軽いフィードバック、ページ遷移のトランジションを検討
- **アクセシビリティ:** コントラスト比（WCAG AA以上）の確保、フォーカス表示の明確化
